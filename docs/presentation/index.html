The problem

RV setup is a coordination problem disguised as a checklist

  • Tasks must happen in the right order, not just eventually.
  • Paper lists and memory fail when one person cannot see what changed.
  • Miscommunication creates rework, delays, and avoidable mistakes.

Tech deep dive

SignalR is the trust layer for shared progress

  • Users join a checklist-specific group: checkset-{id}.
  • Changes broadcast only to the relevant collaborators.
  • Join events help the UI feel live, social, and safe to trust.
Why it matters

If one partner checks off a step, the other should never wonder whether their screen is stale.

public class CheckListHub : Hub<ICheckListHubClient>
{
    public async Task JoinCheckSet(int checkSetId, string userName)
    {
        var groupName = $"checkset-{checkSetId}";
        await Groups.AddToGroupAsync(Context.ConnectionId, groupName);
        await Clients.OthersInGroup(groupName)
            .UserJoined(userName, checkSetId);
    }
}
Tech deep dive

Data model and auth support safe sharing

EF Core + schema discipline

  • Database objects live under the [CheckList] schema.
  • Entities include AppUser, CheckSet, TemplateSet, CheckAction, and CheckSetShare.
  • Sharing is explicit: SharedWithUserId + role-based relationship data.

Microsoft Entra ID

  • Authentication uses Microsoft.Identity.Web.
  • AppUser uses the Entra object ID as the primary key.
  • Identity becomes the anchor for invitations, sharing, and auditability.
Result: a clean path from enterprise identity to checklist collaboration.
Stress that the sharing model is not bolted on. It is represented directly in the core entities and keyed to Entra identity.
